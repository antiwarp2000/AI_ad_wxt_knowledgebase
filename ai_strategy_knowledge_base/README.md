# 万相台 AI 投放策略知识库

这套知识库采用轻量、稳定、容易维护的方式，不依赖向量索引。它的核心不是“把资料全部丢给大模型”，而是让 AI 按固定路径完成广告投放诊断。

## 核心逻辑

```text
算指标 -> 套规则 -> 做归因 -> 查策略文档 -> 参考案例 -> 给动作 -> 复盘沉淀
```

## 设计原则

1. **指标字典负责怎么算**：统一 ROI、CTR、CVR、CPC、CPA、预算消耗率等口径。
2. **规则库负责怎么判**：先用确定性规则判断计划属于黄金、潜力、止损、问题哪一类。
3. **决策树负责为什么**：按异常链路归因，例如无展现、无点击、有点击无转化、ROI 下滑、花费过快、花费过慢。
4. **策略文档库负责怎么做**：保存 SOP、策略说明、经验文档，供人和 AI 按目录/文件名直接查阅。
5. **案例库负责沉淀经验**：记录真实投放动作和结果，让后续判断越来越贴近你的店铺。
6. **动作剧本负责可执行**：把诊断结果转成运营当天能执行的动作清单。

## 目录说明

| 目录 | 用途 | 给 AI 的角色 |
|---|---|---|
| `00_raw_sources` | 原始知识库资料归档 | 保留原文，不直接改 |
| `01_metrics_dictionary` | 指标定义、公式、字段映射 | 统一计算口径 |
| `02_rules_engine` | 计划分层、健康度、阈值规则 | 做确定性判断 |
| `03_decision_trees` | 异常归因路径 | 找问题根因 |
| `04_strategy_docs` | SOP、策略、经验文档 | 做解释和补充建议 |
| `05_case_library` | 历史诊断、动作、结果 | 沉淀经验 |
| `06_action_playbooks` | 不同诊断下的动作 SOP | 生成可执行动作 |
| `07_agent_io_schema` | 输入输出 JSON 结构 | 对接看板/AI Agent |
| `08_review_logs` | 每日复盘与人工校准 | 迭代规则阈值 |

## 推荐调用顺序

```text
广告计划数据
  -> 指标计算
  -> 规则库分层
  -> 决策树归因
  -> 按诊断标签读取策略文档
  -> 案例库匹配历史经验
  -> 输出动作清单
  -> 执行结果写回复盘日志/案例库
```

## 维护原则

- 关键判断放在 `02_rules_engine`，不要只写在长篇文档里。
- 归因路径放在 `03_decision_trees`，让 AI 先找原因再给建议。
- 策略经验放在 `04_strategy_docs`，按主题拆小文档，方便直接读取。
- 真实投放结果放在 `05_case_library`，这是后续越用越准的关键。
- 每次规则调整都记录到 `08_review_logs`，避免改完不知道效果如何。
