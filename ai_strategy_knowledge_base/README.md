# 万相台 AI 投放策略知识库

这套目录用于让广告看板和后续 AI Agent 按稳定逻辑调用策略，不只依赖 RAG 搜索。

## 核心逻辑

广告诊断建议采用：

1. **指标字典**：统一 ROI、CTR、CVR、CPC、CPA、预算消耗率等口径。
2. **规则库**：先用确定性规则判断计划属于黄金、潜力、止损、问题哪一类。
3. **决策树**：再按异常链路归因，例如无展现、无点击、有点击无转化、ROI 下滑、花费过快、花费过慢。
4. **RAG 素材库**：用于补充解释、SOP、策略背景和话术，不直接做最终判断。
5. **案例库**：记录历史动作和结果，让系统逐步沉淀“什么动作有效”。
6. **动作剧本**：把诊断结果转成运营可以执行的动作清单。

## 目录说明

| 目录 | 用途 | 给 AI 的角色 |
|---|---|---|
| `00_raw_sources` | 原始知识库资料归档 | 保留原文，不直接改 |
| `01_metrics_dictionary` | 指标定义、公式、字段映射 | 统一计算口径 |
| `02_rules_engine` | 计划分层、健康度、阈值规则 | 做确定性判断 |
| `03_decision_trees` | 异常归因路径 | 找问题根因 |
| `04_rag_corpus` | 可检索策略文本 | 做解释和补充建议 |
| `05_case_library` | 历史诊断、动作、结果 | 沉淀经验 |
| `06_action_playbooks` | 不同诊断下的动作 SOP | 生成可执行动作 |
| `07_agent_io_schema` | 输入输出 JSON 结构 | 对接看板/AI Agent |
| `08_review_logs` | 每日复盘与人工校准 | 迭代规则阈值 |

## 推荐调用顺序

```text
广告计划数据
  -> 指标计算
  -> 规则库分层
  -> 决策树归因
  -> RAG 检索策略说明
  -> 案例库匹配历史经验
  -> 输出动作清单
  -> 执行结果写回复盘日志/案例库
```

## 维护原则

- 规则库里的判断要尽量结构化、可计算、可复现。
- RAG 文档适合放 SOP、解释、经验、注意事项，不要把关键判断只写在长文本里。
- 案例库每次记录动作前后的数据变化，后续才能判断策略是否有效。
- 阈值不要写死在代码里，优先放在规则配置中，方便按类目和店铺阶段调整。
