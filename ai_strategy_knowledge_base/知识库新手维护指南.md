# 万相台 AI 策略知识库新手维护指南

这份指南给“不懂知识库基础”的人使用。你只需要记住一句话：

> 知识库不是把资料全塞进去，而是把资料放到正确的位置，让 AI 知道什么时候该算、什么时候该判断、什么时候该查资料、什么时候该参考案例。

---

## 1. 先理解这套知识库在干什么

广告投放诊断不能只靠 RAG，因为 RAG 只是“找资料”。真正要做出靠谱诊断，需要四层能力：

```text
算指标 -> 套规则 -> 找原因 -> 给动作
```

对应到当前目录：

| 目录 | 你可以理解成 | 什么时候改它 |
|---|---|---|
| `01_metrics_dictionary` | 指标说明书 | 新增字段、新报表、指标口径变化时 |
| `02_rules_engine` | 判断标准 | 黄金/潜力/止损/问题计划分错时 |
| `03_decision_trees` | 排查路径 | 想让 AI 更会找原因时 |
| `04_rag_corpus` | 策略资料库 | 有新的 SOP、经验、投放方法时 |
| `05_case_library` | 历史案例本 | 每次做完调整并看到结果后 |
| `06_action_playbooks` | 动作清单 | 想规范“诊断后该怎么做”时 |
| `07_agent_io_schema` | 数据接口说明 | 后续接系统、接 API、接 Agent 时 |
| `08_review_logs` | 每日复盘记录 | 每天复盘和校准规则时 |

---

## 2. 最推荐的新手维护流程

每次你想优化知识库，按这个顺序做：

### 第一步：先记录问题

不要一上来就改知识库，先问自己：

- AI 哪个判断不准？
- 是分层错了，还是原因找错了？
- 是建议太泛，还是没有结合你的店铺情况？
- 是缺少案例，还是缺少规则？

### 第二步：判断该改哪个文件夹

| 你遇到的问题 | 应该改哪里 |
|---|---|
| 黄金/潜力/止损/问题分错 | `02_rules_engine` |
| ROI、CTR、CVR 这些算得不对 | `01_metrics_dictionary` |
| AI 不知道为什么出问题 | `03_decision_trees` |
| AI 解释太空泛 | `04_rag_corpus` |
| AI 不知道你以前怎么处理过 | `05_case_library` |
| AI 给的动作不可执行 | `06_action_playbooks` |

### 第三步：小步修改，不要一次大改

每次只改一个点，例如：

- 只调整“止损计划”的 ROI 阈值。
- 只新增一个“有点击无转化”的案例。
- 只补充一个“ROI 下滑”的排查原因。

这样后续如果效果变差，容易知道是哪次修改造成的。

## 3. 具体怎么优化每个模块

## 3.1 优化指标字典

位置：`01_metrics_dictionary/metrics_dictionary.md`

适合新增：

- 新报表字段对应关系。
- 新指标公式。
- 指标是否可为空。
- 花费为 0、点击为 0、订单为 0 时怎么处理。

示例：

```text
新增指标：收藏加购率
公式：收藏加购数 ÷ 点击量
用途：判断新品是否有潜力
注意：点击量为 0 时不计算
```

## 3.2 优化规则库

位置：`02_rules_engine/plan_tier_rules.yaml`

规则库决定 AI 怎么分层。这里尽量写成清晰的“如果...那么...”逻辑。

示例：

```yaml
止损计划:
  any:
    - roi < target_roi * 0.5
    - spend > 0 and orders == 0
```

如果你发现 AI 把太多计划分到止损，可以降低敏感度，比如改成：

```yaml
止损计划:
  any:
    - roi < target_roi * 0.4
    - spend > 100 and orders == 0
```

新手建议：

- 不要频繁改很多阈值。
- 每次改完观察 2-3 天。
- 阈值要结合你的类目、毛利、目标 ROI。

## 3.3 优化决策树

位置：`03_decision_trees/diagnosis_decision_tree.md`

决策树用于回答：为什么这个计划有问题？

比如“有点击无转化”，可以继续细分：

```text
有点击无转化
├─ 流量不精准
├─ 价格没有优势
├─ 详情页承接弱
├─ 评价/销量弱
└─ 竞品活动更强
```

你后续越懂业务，就越应该把原因写细。

## 3.4 优化 RAG 素材库

位置：`04_rag_corpus`

这里放适合 AI 检索的长文本，比如：

- 投放 SOP
- 优化方法
- 活动策略
- 类目经验
- 平台规则
- 合规注意事项

写 RAG 文档时注意：

- 标题要明确，例如“有点击无转化怎么排查”。
- 一篇文档只讲一个主题。
- 多用小标题和表格。
- 不要把关键判断只藏在大段文字里。

推荐格式：

```markdown
# 有点击无转化排查 SOP

## 适用场景

点击量正常，但订单数低或为 0。

## 常见原因

| 原因 | 判断方式 | 动作 |
|---|---|---|
| 流量不精准 | CTR可以但CVR低 | 收窄人群/关键词 |
| 价格弱 | 同款竞品更便宜 | 增加优惠或调整价格 |

## 今日动作

1. 暂停高花费无成交人群。
2. 检查详情页首屏和评价。
3. 观察次日 CVR 是否改善。
```

## 3.5 优化案例库

位置：`05_case_library`

案例库是最重要的长期资产。因为它记录的是“你自己店铺里真实发生过什么”。

每个案例最少记录 5 件事：

1. 当时是什么问题。
2. 当时数据是什么样。
3. 做了什么动作。
4. 第二天/三天后结果如何。
5. 这个经验下次能不能复用。

最简单案例格式：

```markdown
# case_20260707_001：有点击无转化后暂停低效人群

- 计划分层：止损计划
- 异常类型：有点击无转化
- 调整前：花费 300，成交 0，点击 120
- 执行动作：暂停两个高花费无成交人群，优化商品首图
- 复盘结果：次日花费下降 40%，ROI 恢复到 目标 ROI 的 0.9 倍
- 经验：该类商品低价泛人群容易吃预算，后续优先收窄人群
```

## 3.6 优化动作剧本

位置：`06_action_playbooks/action_playbooks.md`

动作剧本负责把诊断翻译成“今天运营该做什么”。

好的动作建议应该是：

- 明确
- 可执行
- 有幅度
- 有复盘时间

不好的建议：

```text
优化投放效果。
```

好的建议：

```text
将该计划预算下调 30%，暂停过去 3 天高花费无成交人群，明天观察 CVR 和 ROI 是否恢复。
```

## 4. 新手最容易犯的 5 个错误

## 错误 1：把所有资料都放进 RAG

RAG 适合找资料，不适合做关键判断。判断标准要放规则库。

## 错误 2：只写经验，不写触发条件

例如“ROI 低要暂停”太粗。应该写：

```text
当 ROI < 目标 ROI * 0.5 且花费超过最低样本阈值时，进入止损计划。
```

## 错误 3：没有案例复盘

没有复盘，AI 永远不知道哪个动作真的有效。

## 错误 4：规则太复杂

新手一开始不要追求完美规则。先用简单规则跑起来，再逐步校准。

## 错误 5：没有版本记录

每次改规则，建议在 `08_review_logs` 里记录：

- 改了什么
- 为什么改
- 观察几天
- 结果是否变好

## 5. 每周优化节奏建议

## 每天做

- 看昨日计划分层是否合理。
- 把明显误判的计划记下来。
- 对重点计划写一条复盘记录。

## 每周做

- 汇总误判最多的类型。
- 微调一次规则阈值。
- 新增 3-5 个真实案例。
- 补充 1-2 篇 SOP 文档。

## 每月做

- 检查哪些规则长期不准。
- 按类目/商品阶段拆更细规则。
- 把高频有效动作沉淀进动作剧本。
- 删除或归档过时策略。

## 6. 你可以怎么给 AI 下指令来维护知识库

## 新增案例

```text
把下面这个投放调整过程整理成案例，放到 case_library：
计划A昨天花费300无成交，我暂停了两个泛人群，今天ROI恢复到5.2。
```

## 优化规则

```text
最近止损计划误判太多，请检查 plan_tier_rules.yaml，帮我把有花费无成交规则改得更稳健，要考虑最低花费阈值。
```

## 补充 SOP

```text
帮我新增一篇“ROI下滑排查SOP”，放到 RAG 素材库，要求按 CPC、CVR、客单价三个因素拆解。
```

## 复盘校准

```text
根据最近7天诊断结果，帮我总结哪些规则误判最多，并提出规则优化建议。
```

## 7. 最简单的判断口诀

```text
指标字典：先算清楚
规则库：先判类型
决策树：再找原因
RAG：补充解释
案例库：沉淀经验
动作库：指导执行
复盘日志：持续校准
```

如果你不知道一个内容该放哪里，就问：

- 它是公式吗？放指标字典。
- 它是判断标准吗？放规则库。
- 它是排查路径吗？放决策树。
- 它是长篇经验/SOP吗？放 RAG。
- 它是真实发生过的调整吗？放案例库。
- 它是运营动作吗？放动作剧本。
- 它是当天复盘吗？放复盘日志。
