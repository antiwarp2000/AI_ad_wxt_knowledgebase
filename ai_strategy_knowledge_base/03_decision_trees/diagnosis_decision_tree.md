# 异常诊断决策树

## 总入口

```text
计划异常
├─ 无展现
├─ 有展现无点击
├─ 有点击无转化
├─ ROI 下滑
├─ 花费过快
└─ 花费过慢
```

## 1. 无展现

触发条件：`impressions == 0`

排查顺序：

1. 计划状态：暂停、审核中、余额不足、商品下架。
2. 预算：预算过低或已撞线。
3. 出价：低于市场竞争水平。
4. 定向：人群过窄、关键词过窄、地域/时段限制过严。
5. 商品：违规、库存不足、标题属性不完整。

动作建议：启用计划、充值、提高出价 10%-20%、扩大定向、恢复商品状态。

## 2. 有展现无点击

触发条件：`impressions > 0 and clicks == 0` 或 `ctr < ctr_baseline * 0.7`

排查顺序：

1. 创意：主图吸引力、标题卖点、价格利益点。
2. 相关性：关键词/人群与商品是否匹配。
3. 竞争力：价格、评价、销量、优惠是否弱于竞品。
4. 素材合规：是否被降权或审核影响。

动作建议：更换主图/标题、暂停不相关词、增加优惠表达、优化商品首屏。

## 3. 有点击无转化

触发条件：`clicks > 0 and orders == 0` 或 `cvr < cvr_baseline * 0.7`

排查顺序：

1. 流量精准度：关键词、人群、场景是否偏泛。
2. 商品承接：详情页、评价、价格、SKU、库存。
3. 竞品影响：竞品促销、排名、价格优势。
4. 转化路径：优惠券、客服、下单链路是否顺畅。

动作建议：收窄人群/关键词，优化详情页和评价，调整优惠，暂停高花费无转化流量。

## 4. ROI 下滑

触发条件：`roi < target_roi * 0.8` 或 ROI 连续下降。

拆解公式：

```text
ROI = 成交金额 / 花费
ROI ≈ 客单价 × 转化率 / CPC
```

排查顺序：

1. CPC 是否上涨。
2. CVR 是否下降。
3. 客单价是否下降。
4. 成交金额是否受活动、库存、价格影响。
5. 是否出现低质流量扩量。

动作建议：控制 CPC、优化转化承接、提高客单价、缩减低效流量、恢复高 ROI 结构。

## 5. 花费过快

触发条件：预算消耗率过高或早段时间快速撞线。

排查顺序：出价过高、定向过宽、匹配过宽、时段未控制、预算过低。

动作建议：降低出价 10%-20%、收窄定向、控制时段、增加预算但仅限 ROI 健康计划。

## 6. 花费过慢

触发条件：预算消耗率低、展现/点击不足。

排查顺序：出价偏低、定向过窄、预算过低导致系统不学习、商品权重弱。

动作建议：提高出价 10%-20%、扩大人群/关键词、补足预算、优化商品基础权重。
