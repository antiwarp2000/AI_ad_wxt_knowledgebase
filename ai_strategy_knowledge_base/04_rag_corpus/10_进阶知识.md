## 10.1 达摩盘人群深度应用

### 人群标签体系
| 标签类型 | 示例 | 应用场景 |
|---------|------|---------|
| 基础属性 | 性别、年龄、地域、消费层级 | 基础定向 |
| 行为兴趣 | 浏览/收藏/加购/购买行为 | 精准定向 |
| 商品偏好 | 品类偏好、价格偏好、品牌偏好 | 商品匹配 |
| 生命周期 | 新客、老客、沉睡客、流失客 | 分层运营 |

### Lookalike扩展原理
```
Lookalike扩展逻辑：
1. 选择种子人群(如已购客户)
2. 系统分析种子人群特征
3. 在全量用户中寻找相似特征用户
4. 按相似度排序输出扩展人群

相似度设置：
- 1%：最精准，覆盖小，转化高
- 3%：平衡型，覆盖中，转化中
- 5%：覆盖大，转化略低
```

### 人群包管理
| 操作 | 频率 | 说明 |
|------|------|------|
| 人群包更新 | 每周 | 确保人群时效性 |
| 人群效果评估 | 每周 | 对比各人群包ROI |
| 人群包替换 | 每月 | 替换低效人群包 |
| 人群资产分析 | 每月 | 分析AIPL流转率 |

## 10.2 智能出价算法理解

### 目标ROI出价算法
```
算法逻辑：
1. 系统根据历史数据预测每次点击的转化概率
2. 根据目标ROI反推可接受的最高CPC
3. 实时竞价中动态调整出价
4. 最终使整体ROI接近目标值

数据要求：
- 至少15个转化数据
- 投放时间≥7天
- 预算充足(日预算≥50个点击成本)
```

### 最大化拿量算法
```
算法逻辑：
1. 在预算约束下，尽可能获取更多转化
2. 不严格控制单次转化成本
3. 适合预算充足、追求规模的场景

适用条件：
- 预算充足
- 不严格控成本
- 追求转化量最大化
```

### 控成本投放算法
```
算法逻辑：
1. 严格控制单次转化成本≤设定值
2. 当CPA接近上限时降低出价
3. 可能导致拿量不足

适用条件：
- 成本敏感
- 预算有限
- 需要严格控制CPA
```

## 10.3 多渠道协同投放

### 搜索+推荐协同
| 渠道 | 特点 | 协同策略 |
|------|------|---------|
| 搜索 | 意图明确、转化高 | 核心词精确匹配，高ROI |
| 推荐 | 流量大、成本低 | 人群定向，扩量 |
| 协同 | - | 搜索打转化，推荐扩规模 |

### 关键词推广+人群推广配合
| 组合 | 策略 | 说明 |
|------|------|------|
| 关键词为主 | 关键词获取精准流量，人群辅助溢价 | 适合搜索流量为主的类目 |
| 人群为主 | 人群获取推荐流量，关键词辅助 | 适合推荐流量为主的类目 |
| 均衡 | 关键词+人群各占50%预算 | 适合全渠道布局 |

### 万相台+直通车+引力魔方组合
| 平台 | 定位 | 预算占比 |
|------|------|---------|
| 万相台 | 智能投放、全渠道 | 50%-60% |
| 直通车 | 搜索精准流量 | 20%-30% |
| 引力魔方 | 推荐流量 | 10%-20% |

## 10.4 数据归因与分析

### 归因模型
| 模型 | 规则 | 适用场景 |
|------|------|---------|
| 末次点击 | 转化归因于最后一次点击 | 默认模型，简单直接 |
| 首次点击 | 转化归因于第一次点击 | 评估拉新效果 |
| 线性归因 | 转化均匀分配给所有点击 | 评估全链路贡献 |
| 时间衰减 | 越接近转化的点击权重越高 | 平衡首次和末次 |

### 归因周期
| 周期 | 说明 | 适用场景 |
|------|------|---------|
| 1天 | 仅统计点击后1天内转化 | 即时转化商品 |
| 7天 | 统计点击后7天内转化 | 常规商品 |
| 15天 | 统计点击后15天内转化 | 决策周期较长商品 |
| 30天 | 统计点击后30天内转化 | 高客单/长决策商品 |

### 自然流量与付费流量协同
```
协同效应：
- 付费流量提升商品权重 → 自然流量增加
- 自然流量转化好 → 付费流量ROI提升
- 付费+自然总ROI > 单独付费ROI

评估方法：
- 对比投放前后自然流量变化
- 计算付费+自然总ROI
- 分析付费流量对自然流量的带动系数
```
